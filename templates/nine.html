#9 Using Components with Known Vulnerabilities

This category of threats are applicable to all those websites that have a lot of dependencies or have legacy module dependant structure which may cause hinderance to regular updating cycles. Failing to update your system dependencies may result in the web application running outdated modules with less stringent security provisions thereby making them easier targets for malicious attacks. There are many reasons why an organization may be unable to update their application dependencies as and when they roll out. Two of the most common ones are: (1) Web masters and developers cannot keep up with the pace of updates as updating properly does take a lot of time and (2) Legacy code does not work on newer versions of its dependencies. Whatever may be the reason for not having up to date modules, this decision allows unknown vulnerabilities to persist in the system which may be manipulated by cyber criminals to launch attacks.

In cases where it is not possible to run the most up to date software dependencies, OWASP recommends using virtual patching for the cases where patching up is not possible. This technique allows outdated websites to be protected from attacks by using firewalls and intrusion detection systems.

According to OWASP, some of the ways to prevent suing components with known vulnerabilities are: Remove all unnecessary dependencies. Have an inventory of all your components on the client-side and server-side. Monitor sources like Common Vulnerabilities and Disclosures (CVE) and National Vulnerability Database (NVD) for vulnerabilities in the components. Obtain components only from official sources. Get rid of components not actively maintained and use virtual patching.
